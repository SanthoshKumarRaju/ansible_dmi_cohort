History & Evolution

The journey of Infrastructure as Code and configuration management tools

2012

Ansible Founded

Michael DeHaan created Ansible, naming it after the fictional faster-than-light communication system from Orson Scott Card's Ender's Game

  • First release: Ansible 0.0.1
  • Agentless architecture
  • YAML-based playbooks
2014

Terraform Born

HashiCorp introduced Terraform as the first multi-cloud infrastructure provisioning tool

  • Declarative configuration language
  • State management system
  • Resource graph visualization
2015

Red Hat Acquisition

Red Hat acquired Ansible, accelerating its enterprise adoption and development

  • Ansible Tower introduced
  • Enterprise features added
  • Community growth
2020+

Modern Era

Both tools became industry standards with extensive ecosystems

  • Terraform 1.0 released
  • Ansible Collections introduced
  • Cloud-native integrations

Complete Terraform Example 
# main.tf - Complete AWS Infrastructure
terraform {
  required_version = ">= 1.0"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }

  backend "s3" {
    bucket = "my-terraform-state"
    key    = "prod/terraform.tfstate"
    region = "us-east-1"
  }
}

# Provider Configuration
provider "aws" {
  region = var.aws_region
  profile = "default"
}

# Variables
variable "aws_region" {
  description = "AWS region"
  type        = string
  default     = "us-east-1"
}

variable "instance_type" {
  description = "EC2 instance type"
  type        = string
  default     = "t3.micro"
}

# Data Sources
data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"] # Canonical

  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server-*"]
  }
}

# VPC Resources
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = {
    Name = "main-vpc"
    Environment = "production"
  }
}

# Internet Gateway
resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name = "main-igw"
  }
}

# Public Subnet
resource "aws_subnet" "public" {
  vpc_id                  = aws_vpc.main.id
  cidr_block              = "10.0.1.0/24"
  availability_zone       = "us-east-1a"
  map_public_ip_on_launch = true

  tags = {
    Name = "public-subnet"
  }
}

# Route Table
resource "aws_route_table" "public" {
  vpc_id = aws_vpc.main.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.main.id
  }

  tags = {
    Name = "public-route-table"
  }
}

# Security Group
resource "aws_security_group" "web_sg" {
  name        = "web-security-group"
  description = "Allow web traffic"
  vpc_id      = aws_vpc.main.id

  ingress {
    description = "HTTP"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "HTTPS"
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "SSH"
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Name = "web-sg"
  }
}

# EC2 Instance
resource "aws_instance" "web_server" {
  ami                    = data.aws_ami.ubuntu.id
  instance_type          = var.instance_type
  subnet_id              = aws_subnet.public.id
  vpc_security_group_ids = [aws_security_group.web_sg.id]
  key_name               = "my-keypair"

  user_data = <<-EOF
    #!/bin/bash
    apt-get update
    apt-get install -y nginx
    systemctl start nginx
    systemctl enable nginx
    echo "<h1>Hello from Terraform!</h1>" > /var/www/html/index.html
  EOF

  root_block_device {
    volume_size = 20
    volume_type = "gp3"
  }

  tags = {
    Name        = "web-server"
    Environment = "production"
    Project     = "terraform-demo"
  }
}

# Outputs
output "web_server_public_ip" {
  description = "Public IP of web server"
  value       = aws_instance.web_server.public_ip
}

output "web_server_public_dns" {
  description = "Public DNS of web server"
  value       = aws_instance.web_server.public_dns
}

output "vpc_id" {
  description = "VPC ID"
  value       = aws_vpc.main.id
}

Ansible Comprehensive Guide

Agentless automation for configuration management and application deployment

Architecture & Components

Ansible follows a simple, agentless architecture that uses SSH or WinRM to connect to nodes and execute tasks defined in YAML playbooks.

Control Node

Machine running Ansible with inventory and playbooks

→

SSH/WinRM

Agentless communication protocol

→

Managed Nodes

Target systems being configured

Core Components

Inventory

Lists of managed hosts organized in groups

Playbooks

YAML files defining automation tasks and workflows

Modules

Reusable units of work (package management, files, services)

Roles

Collections of related playbooks, files, and templates

Complete Ansible Playbook 
---
# site.yml - Complete Web Application Deployment
- name: Configure web servers
  hosts: webservers
  become: yes
  vars:
    app_name: "my-webapp"
    web_root: "/var/www/{{ app_name }}"
    db_host: "{{ groups['dbservers'][0] }}"
    domain_name: "example.com"
    
  pre_tasks:
    - name: Update package cache
      apt:
        update_cache: yes
        cache_valid_time: 3600
      when: ansible_os_family == 'Debian'

  roles:
    - role: nginx
    - role: nodejs
    - role: postgresql

  tasks:
    - name: Create application user
      user:
        name: "{{ app_name }}"
        system: yes
        create_home: yes
        shell: /bin/bash

    - name: Create web directory structure
      file:
        path: "{{ item }}"
        state: directory
        owner: "{{ app_name }}"
        group: "{{ app_name }}"
        mode: '0755'
      loop:
        - "{{ web_root }}"
        - "{{ web_root }}/logs"
        - "{{ web_root }}/config"

    - name: Deploy application code
      git:
        repo: "https://github.com/example/{{ app_name }}.git"
        dest: "{{ web_root }}/app"
        version: "main"
        update: yes

    - name: Install Node.js dependencies
      npm:
        path: "{{ web_root }}/app"
        production: yes
      environment:
        NODE_ENV: production

    - name: Configure nginx virtual host
      template:
        src: templates/nginx.conf.j2
        dest: /etc/nginx/sites-available/{{ app_name }}
        mode: '0644'
      notify: restart nginx

    - name: Enable nginx site
      file:
        src: /etc/nginx/sites-available/{{ app_name }}
        dest: /etc/nginx/sites-enabled/{{ app_name }}
        state: link
      notify: restart nginx

    - name: Setup application environment
      template:
        src: templates/.env.j2
        dest: "{{ web_root }}/app/.env"
        mode: '0644'

    - name: Install PM2 process manager
      npm:
        name: pm2
        global: yes
        production: yes

    - name: Start application with PM2
      command:
        cmd: "pm2 start {{ web_root }}/app/server.js --name {{ app_name }}"
        chdir: "{{ web_root }}/app"
      environment:
        NODE_ENV: production

    - name: Setup PM2 startup script
      command: pm2 startup
      ignore_errors: yes

    - name: Setup application monitoring
      cron:
        name: "Check application health"
        minute: "*/5"
        job: "curl -f http://localhost:3000/health || pm2 restart {{ app_name }}"

  handlers:
    - name: restart nginx
      service:
        name: nginx
        state: restarted
        enabled: yes

    - name: reload nginx
      service:
        name: nginx
        state: reloaded

    - name: restart application
      command: pm2 restart {{ app_name }}

- name: Configure database servers
  hosts: dbservers
  become: yes
  vars:
    postgresql_version: 14
    databases:
      - name: "{{ app_name }}"
        owner: "{{ app_name }}_user"

  tasks:
    - name: Install PostgreSQL
      apt:
        name: "postgresql-{{ postgresql_version }}"
        state: present

    - name: Ensure PostgreSQL is running
      service:
        name: "postgresql@{{ postgresql_version }}-main"
        state: started
        enabled: yes

    - name: Create application database
      postgresql_db:
        name: "{{ item.name }}"
        owner: "{{ item.owner }}"
      loop: "{{ databases }}"

    - name: Create database user
      postgresql_user:
        name: "{{ app_name }}_user"
        password: "{{ vault_db_password }}"
        encrypted: yes
        role_attr_flags: "CREATEDB,LOGIN"

- name: Deploy to production
  hosts: webservers:&dbservers
  tasks:
    - name: Send deployment notification
      slack:
        token: "{{ slack_token }}"
        msg: "Deployment of {{ app_name }} completed successfully!"
        channel: "#deployments"
      delegate_to: localhost
      run_once: yes

Essential Commands & Workflows

Master the command-line interface for both tools

Terraform Commands

terraform init

Initialize working directory with providers and modules

terraform plan

Preview changes before applying

terraform apply

Apply changes to reach desired state

terraform destroy

Destroy all managed infrastructure

terraform fmt

Format configuration files

terraform validate

Validate configuration syntax

terraform state list

List resources in state file

terraform output

Show output values

Ansible Commands

ansible-playbook playbook.yml

Run an Ansible playbook

ansible all -m ping

Test connectivity to all hosts

ansible-doc module_name

Show documentation for a module

ansible-vault create file.yml

Create encrypted file

ansible-galaxy init role_name

Initialize new role structure

ansible-inventory --list

Show inventory in JSON format

ansible-playbook --check

Dry run without making changes

ansible-playbook --syntax-check

Check playbook syntax
